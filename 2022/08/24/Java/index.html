Java8 中optional的用法

引入optional的目的

Optional用于缓解Java臭名昭著的空指针异常问题。它的设计意图为：

  • Optional 是用来作为方法返回值的
  • Optional 是为了清晰地表达返回值中没有结果的可能性
  • 且如果直接返回 null 很可能导致调用端产生错误（尤其是NullPointerException)

一些常用的api

创建optional类

  • Optional.of()：of不允许参数是null
  • Optional.ofNullable()：参数无限制
  • Optional.empty()：所有null包装成的Optional对象

判断值是否存在

  • isPresent()

  • isEmpty()

  • ifPresent()

获取optional的值

  • orElse()：如果optional对象保存的值不是null，则返回原来的值，否则返回value

  • orElseGet()：类似于 orElse()，如果没有Optional值，采用供应商功能接口，该接口将被调用并返回调用的值。

  • orElseThrow(): 值不存在则抛出异常
